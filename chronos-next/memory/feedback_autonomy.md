Avanzar directamente con los cambios de código sin pedir permiso para cada edición individual.

**Why:** El usuario lo pidió explícitamente — le resulta lento que pregunte por cada cosa.

**How to apply:** Hacer los cambios necesarios y explicar qué hice después, no antes. Solo preguntar cuando sea algo destructivo o irreversible (borrar datos, cambiar credenciales de producción, etc.).
